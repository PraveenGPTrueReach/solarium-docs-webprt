Features available in this project

A concise, customer‑friendly overview of the capabilities in the Solarium Web Portal. This page is derived from the project’s source and internal documentation and avoids speculative items.

Authentication & Session

Email/password login for Admin and Key Account Manager (KAM). Inactivity timeout is 30 minutes; expired or unauthorized requests trigger a clear sign‑out and re‑login.

Role‑based UI & Territories

Conditional navigation and feature gating based on role. KAM access is scoped by assigned territories; Admin has full access.

Lead Management

Filterable lists, detail views, status updates (with override reason), timeline of changes, bulk operations up to 50 items, and CSV export aligned with current filters.

Quotation Management

Seven‑step wizard with draft save for Admin/KAM, sharing with customers, an explicit “Customer Accepted” stage before “Won,” and immutable PDFs once finalized. Only one accepted quotation per lead is allowed.

Commissions & Payouts

Admin reviews and can override amounts, then marks commissions as Paid (including bulk pay). Paid status is final and not reversible.

Master Data

Admin‑only CRUD for panels, inverters, fees, and related parameters. Changes apply immediately to new or resumed quotations; soft‑delete and basic audit are maintained.

Documents & KYC

View and approval workflows with short‑lived upload/download links. Approved KYC documents cannot be superseded; latest versions are shown while older ones are retained as required. Practical limits include single PUT uploads (~10 MB) and up to seven attachments per lead.

Settings & Feature Flags

Admin can toggle selected settings. Active sessions pick up changes via simple periodic refresh without forcing immediate reloads.

Reporting & Dashboards

Minimal summaries (e.g., leads by status, pending vs. paid) and CSV export for deeper external analysis.

Support Tickets

View ticket threads, reply (including attachments), and close with notes.
